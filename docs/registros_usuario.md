## Historia de Usuario 1 — Gestión de Dotaciones

*Identificador:* HU01  
*Nombre:* Registrar dotaciones de usuarios  

### Historia de Usuario

Como administrador del sistema, quiero registrar las dotaciones de los beneficiarios para llevar un control organizado de las entregas realizadas.

### Criterios de Aceptación

*CA01:*  
El sistema debe permitir ingresar:

- nombre de la dotación
- descripción
- fecha de entrega
- cantidad
- valor

*CA02:*  
El administrador podrá asociar la dotación a un beneficiario.

*CA03:*  
El sistema debe guardar el estado de la dotación.

## Historia de Usuario 6 — Historial de Usuario

*Identificador:* HU06  
*Nombre:* Consultar historial de actividades  

### Historia de Usuario

Como administrador del sistema, quiero visualizar el historial de acciones realizadas por los usuarios para llevar seguimiento y control dentro del sistema.

### Criterios de Aceptación

*CA01:*  
El sistema debe registrar acciones como:

- inicio de sesión
- cierre de sesión
- registro de dotaciones
- actualización de información
- generación de reportes

*CA02:*  
El historial debe mostrar:

- usuario
- acción realizada
- fecha
- hora

*CA03:*  
El administrador podrá consultar el historial de cada usuario.
